## 1. 目標

幫使用者用自己鍾意嘅成人動畫對白做 **shadowing（跟讀）** 練習，提升英文口語、發音同節奏。

核心價值：
- **唔使自己揾片** —— app 自動搜尋劇集 clip 做素材
- **即時 feedback** —— 錄低自己跟讀，轉文字同原句逐字對比，標出讀錯/漏字
- 全程戴住耳機喺手機度練

成功標準（v1）：可以揀一套劇 → app 自動搵到有字幕嘅 clip → 切句 → 逐句播原音、錄音、對比、並排重聽 → 記錄練習進度。

---

## 2. 平台與部署

- **PWA**（Progressive Web App）：網頁技術寫，可「加到手機主畫面」當 app 用。
- **前端**：原生 HTML + JS + CSS（唔用 React）。瀏覽器 `MediaRecorder` 錄音、`<audio>` 播放。
- **後端**：Python FastAPI，跑喺使用者部電腦；手機連到去用。
- 重活（下載、轉文字、切句、對比）全部喺後端做。
- 全部本機跑，**唔使任何收費 API、唔使開戶口**。

```
[手機/電腦瀏覽器  ← PWA 前端]
        │  HTTP / 上載錄音
        ▼
[部電腦  ← Python 後端 FastAPI]
   ├─ yt-dlp        自動 search + 抽 YouTube 音檔 + 字幕
   ├─ faster-whisper  (a) 字幕唔齊時轉原音；(b) 轉使用者跟讀錄音
   ├─ 切句 + 對齊    將音檔切成一句一句、配時間碼
   └─ 對比引擎       使用者文字 vs 原句 → 標讀錯/漏字 (difflib)
        │
        ▼
   SQLite           存素材、句子、練習記錄
```

---

## 3. 核心使用循環

1. 使用者揀一套劇（或撳「隨機」）
2. 後端用 yt-dlp 自動 search YouTube clip，篩走冇英文字幕嘅
3. 抽音 + 字幕 → 切成一句一句 → 入練習庫
4. 揀一句 → **▶ 播原句**（可 0.75x 慢放、可 loop）
5. **● 錄跟讀** → 上載
6. 後端 Whisper 轉錄音 → 同原句**逐字對比** → 標綠（啱）/ 紅（錯漏）
7. **並排播放**：原句 vs 自己把聲，用耳朵再 check
8. 標「過咗 / 再練」→ 入練習記錄 → 跳下一句

| 部件 | 揀咩 | 點解 |

## 5. 素材：自動搜尋

- **內建劇集清單**（v1）：BoJack Horseman、Simpsons、Rick and Morty、Family Guy、South Park、Bob's Burgers。日後可加。
- 流程：揀劇 → 後端 `ytsearch:"<劇名> best scenes"` → **篩走冇英文字幕嘅片** → 優先「有英文字幕、長度 1–5 分鐘、對白為主、背景音樂唔重」嘅 clip → 抽音 + 字幕 → 切句。
- 使用者**全程唔使貼任何 link**。

**法律/使用範圍**：自動下載 YouTube 版權片段僅供**個人學習、離線練習**用，唔作公開再分享。Netflix/Disney+ 完整集數有 DRM，抽唔到，唔做。

---

## 6. 錯誤處理

- **搵唔到合適片**（冇字幕/太短）→ 自動試下一個搜尋結果；連續失敗 → 友善提示「呢套劇暫時搵唔到啱嘅片，試另一套」。
- **yt-dlp 抽片失敗**（片被刪/地區限制）→ skip，試下一條，唔好成個 app 死。
- **Whisper 第一次 load model 慢** → 前端顯示「處理緊…」進度，唔好黑屏。
- **錄音冇咪權限** → 清楚提示開瀏覽器咪權限。
- **字幕無標點切句切亂** → 用 Whisper 時間碼 + 停頓做後備切法。

---

## 7. 測試方向

採用 **TDD**：核心邏輯先寫 test 再寫實作。

- **對比引擎（心臟）** → 單元測試：餵多組「原句 vs 講出嚟嘅文字」，check 標啱錯漏字。
- **切句邏輯** → 餵一段字幕，check 句子數同時間碼合理。
- **yt-dlp / Whisper** → 用 mock 或一條短樣本片行 end-to-end，唔使每次落網絡。
